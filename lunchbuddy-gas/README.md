# 🍱 런치버디 (LunchBuddy) — 배포 가이드

구글 시트 + 구글 앱스크립트만으로 동작하는 사내 점심 멤버 확인 서비스입니다.
(요구사항 명세서 v1.1 기준 구현)

## 폴더 구성

| 파일 | 내용 | 붙여넣을 위치 |
|------|------|---------------|
| `Code.gs` | 백엔드 전체 (인증, 방, 참여, 채팅, 히스토리, 트리거) | 앱스크립트 `Code.gs` |
| `index.html` | 프론트엔드 전체 (단일 페이지 웹앱) | 앱스크립트 HTML 파일 `index` |
| `appsscript.json` | 프로젝트 설정 (시간대, 웹앱 권한) | 앱스크립트 `appsscript.json` |

---

## 배포 순서 (약 10분)

### 1. 구글 시트 만들기
1. [sheets.new](https://sheets.new) 접속 → 새 스프레드시트 생성
2. 이름을 `런치버디DB` 등으로 변경
3. ⚠️ 이 시트가 유일한 데이터 저장소입니다. **다른 사람과 공유하지 마세요** (NFR-04)

### 2. 앱스크립트 열고 코드 붙여넣기
1. 시트 메뉴 **확장 프로그램 > Apps Script** 클릭
2. 기본 `Code.gs` 내용을 전부 지우고 이 폴더의 **`Code.gs`** 내용 붙여넣기
3. 좌측 파일 목록 **＋ > HTML** 클릭 → 파일 이름 **`index`** 로 생성 → 기본 내용을 지우고 이 폴더의 **`index.html`** 내용 붙여넣기
4. 좌측 **⚙ 프로젝트 설정 > "appsscript.json" 매니페스트 파일을 편집기에 표시** 체크
5. 편집기에 나타난 `appsscript.json`을 이 폴더의 **`appsscript.json`** 내용으로 교체 (시간대 Asia/Seoul 설정)

### 3. 초기 설정 실행
1. 편집기 상단 함수 선택 드롭다운에서 **`setup`** 선택 → **실행**
2. 최초 실행 시 권한 요청 → **권한 검토 > 계정 선택 > 고급 > 이동 > 허용**
3. 실행이 끝나면 시트에 7개 탭(Users, Sessions, Rooms, Members, Responses, History, Messages)이 생기고, 매일 자정 초기화 트리거가 등록됩니다

### 4. 웹 앱 배포
1. 우측 상단 **배포 > 새 배포**
2. 유형 선택(⚙) → **웹 앱**
3. 설정:
   - 실행 계정: **나**
   - 액세스 권한: **모든 사용자** ← 팀원이 구글 로그인 없이 접근하려면 필수
4. **배포** 클릭 → 발급된 **웹 앱 URL**을 팀에 공유

### 5. 사용 시작
1. 웹 앱 URL 접속 → 회원가입 (아이디 + 비밀번호) — URL만 알면 누구나 가입·방 생성 가능
2. 방 만들기 → 동료에게 둘 중 하나 공유:
   - **초대 링크** (7일 유효): 링크 접속 → 가입/로그인 → 자동으로 방 참여
   - **방 코드** (6자리, 만료 없음): 방 목록 화면의 "🔑 방 코드로 참여"에 입력
3. 방 메인 화면에서 초대 링크 복사·방 코드 확인 모두 가능

> ⚠️ 이미 `setup()`을 실행한 상태에서 코드를 새 버전으로 교체했다면, **`setup()`을 한 번 더 실행**하세요.
> Rooms 시트에 roomCode 열이 추가되고 기존 방에도 방 코드가 자동 발급됩니다. (재실행해도 데이터는 안전합니다)

## 운영 가이드

### 비밀번호 초기화 (명세서 9.1 A안)
사용자가 비밀번호를 잊었을 때:
1. 앱스크립트 편집기에서 `Code.gs` 맨 아래 **`adminResetPassword`** 함수의
   `TARGET_LOGIN_ID`(대상 아이디)와 `TEMP_PASSWORD`(임시 비밀번호)를 수정
2. 함수 선택 드롭다운에서 `adminResetPassword` 선택 → 실행
3. 사용자에게 임시 비밀번호를 전달하고, 로그인 후 **설정 > 비밀번호 변경**을 안내

### 코드 수정 후 재배포
- **배포 > 배포 관리 > ✏️ 수정 > 버전: 새 버전 > 배포**
- "새 배포"를 만들면 URL이 바뀌므로, 기존 URL 유지를 위해 반드시 **배포 관리**에서 수정하세요

### 자정 초기화가 안 도는 것 같을 때
- 트리거는 00:00~01:00 사이에 실행되며 ±15분 오차가 있을 수 있습니다
- 응답 데이터는 날짜 키로 저장되므로 트리거가 지연돼도 **당일 화면은 항상 정상**입니다 (전일 데이터의 히스토리 반영만 늦어짐)
- 편집기 좌측 **트리거(⏰)** 메뉴에서 `dailyReset` 트리거 존재 여부 확인, 없으면 `setup()` 재실행

### 데이터 백업 (명세서 9.5 권장)
- 스프레드시트 **파일 > 사본 만들기**를 주 1회 수동 실행하거나,
- 자동화를 원하면 요청 주시면 백업 트리거 함수를 추가해드립니다

## 주요 사양 요약

| 항목 | 값 |
|------|-----|
| 세션 유효기간 | 30일 (localStorage 토큰) |
| 로그인 차단 | 5회 연속 실패 시 10분 차단 |
| 초대 링크 만료 | 생성 후 7일, 재생성 시 이전 링크 즉시 무효화 |
| 방 코드 | 6자리 영구 코드 (만료 없음, 방 삭제 시까지 유지) |
| 현황 갱신 | 30초 자동 폴링 + 수동 새로고침 |
| 채팅 | 텍스트 300자, 최근 100개 표시, 15초 폴링 |
| 데이터 보관 | 참여 기록·채팅 30일 후 자동 삭제 |
| 방 삭제 | 호스트 전용 (멤버·기록·채팅 전체 삭제) |
